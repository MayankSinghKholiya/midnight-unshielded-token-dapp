# UI notes

This frontend avoids copying any competitor interface. It uses general high-performing dApp UX patterns:

- A clear wallet/status card at the top.
- One primary action per operation card.
- Token state visible before forms.
- Disabled/busy states and an activity log.
- A dedicated education panel for the unshielded/shielded tradeoff.
- No Ethereum-specific copy, address validation, or chain terminology.

The visual direction is an original Midnight-style dark interface with aurora gradients, glass panels, strong contrast, and mobile-first responsive layout.
